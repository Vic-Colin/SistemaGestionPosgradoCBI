Documentación JavaScript

Sistema de Gestión de Posgrado CBI - UAM Azcapotzalco

Esta documentación describe los módulos JavaScript que gestionan la interacción del frontend con los servlets del backend mediante peticiones AJAX.

Dependencias Externas:
  • jsPDF v2.5.1 - Generación de documentos PDF
  • jsPDF-AutoTable v3.5.31 - Tablas en PDF

alumnos.js

Propósito: Módulo principal para la gestión completa del alumnado del posgrado. Maneja operaciones CRUD, filtrado dinámico, modales de registro/edición y exportación de reportes.

Funcionalidades Principales:

  • Configuración dinámica de campos: Genera automáticamente los selects de trimestres (ingreso y pierde calidad) basados en el año actual
  • Carga de catálogos: Obtiene áreas de concentración y profesores desde AlumnadoServlet para llenar selects dinámicos
  • Sistema de acordeones: Permite expandir/colapsar secciones opcionales (Beca, Titulación, Tesis)
  • Gestión de modales: Controla apertura/cierre de modales para registro, edición y confirmación de eliminación
  • Renderizado de tabla: Genera filas dinámicamente con formato de estatus UAM (colores por estado)
  • Filtros en tiempo real: Búsqueda con debounce (300ms) por matrícula, trimestre, estatus, año de titulación
  • Edición robusta: Función seleccionarOpcionRobusta() para manejar diferencias de formato en selects
  • Exportación a PDF: Genera reportes en tamaño legal con encabezado institucional (rojo UAM #CD032E)

Servlets Conectados:

Endpoint Método Propósito
../AlumnadoServlet?accion=catalogos GET Obtener áreas y profesores para selects
../AlumnadoServlet?accion=buscar&matricula=XXX GET Cargar datos completos para edición
../AlumnadoServlet GET Consultar alumnos con filtros
../AlumnadoServlet POST Crear, editar o eliminar alumno
Nota: Incluye validación de sesión implícita mediante redirección en caso de timeout.

Etiquetas: CRUD AJAX PDF Filtros Modales

becas.js

Propósito: Módulo especializado en la consulta y seguimiento de becas CONAHCYT/SECIHTI asociadas a alumnos del posgrado.

Funcionalidades Principales:

  • Generación dinámica de generaciones: Crea opciones de trimestres para los últimos 10 años automáticamente
  • Filtrado múltiple: Permite buscar por CVU, generación (trimestre de ingreso) y estatus de beca
  • Debounce optimizado: Espera 300ms después de dejar de escribir para evitar peticiones excesivas
  • Formato condicional: Aplica clases CSS según el estatus de la beca para visualización con colores
  • Manejo de nulos: Función auxiliar val() para mostrar "-" cuando los datos son null
  • Exportación a PDF: Reporte en tamaño carta con encabezado institucional
  • Mensajes de error: Muestra mensaje en la tabla si falla la carga de datos

Servlets Conectados:

Endpoint Método Propósito
../BecaServlet?cvu=XXX&generacion=YY-T&estatus=ZZZ GET Consultar becas con filtros combinados
Nota: Los datos de beca se obtienen mediante JOIN con la tabla alumno para incluir nombre y trimestre de ingreso.

Etiquetas: CONAHCYT Filtros AJAX PDF Debounce

profesorado.js

Propósito: Módulo para la consulta y filtrado del personal académico del Núcleo Académico del Posgrado.

Funcionalidades Principales:

  • Búsqueda dual: Permite filtrar simultáneamente por número económico y CVU
  • Debounce compartido: Función cargarConDelay() que optimiza peticiones para ambos inputs
  • Visualización completa: Muestra 8 columnas con información académica del profesor
  • Manejo de valores opcionales: Usa operador || "-" para campos que pueden ser null
  • Exportación a PDF: Reporte en tamaño carta con estilo institucional
  • Validación de elementos: Verifica existencia de inputs antes de agregar event listeners

Servlets Conectados:

Endpoint Método Propósito
../ProfesoradoServlet?cvu=XXX&numeroEconomico=YYY GET Consultar profesores con filtros
Nota: Los datos incluyen JOINs con tablas nivel_sni, tipo_dedicacion y departamento.

Etiquetas: Profesores SNI Filtros AJAX PDF

proyectos.js

Propósito: Módulo para la consulta de proyectos de tesis e investigación asociados a alumnos del posgrado.

Funcionalidades Principales:

  • Búsqueda tricriterio: Filtra por matrícula, título de tesis o nombre de asesor (director/codirector)
  • Búsqueda OR en asesores: El SQL busca coincidencia en director O codirector
  • Formato de estatus UAM: Función getClaseEstatusUam() para badges de colores
  • Encoding de parámetros: Usa encodeURIComponent() para manejar caracteres especiales en títulos
  • Debounce individual: Cada input tiene su propio temporizador de 300ms
  • Exportación a PDF: Reporte con título destacado en negritas
  • Manejo de "No asignado": Muestra texto en gris cuando no hay codirector

Servlets Conectados:

Endpoint Método Propósito
../ProyectosServlet?matricula=XXX&titulo=YYY&asesor=ZZZ GET Consultar proyectos de tesis con filtros
Nota: Los datos se obtienen mediante múltiples JOINs: tesis, alumno, profesor, area_concentracion, estatus_alumno.

Etiquetas: Tesis Proyectos Filtros AJAX PDF

index.js

Propósito: Módulo de autenticación que gestiona el inicio de sesión y control de acceso al sistema.

Funcionalidades Principales:

  • Prevención de envío tradicional: Usa event.preventDefault() para manejar el formulario vía AJAX
  • Validación de credenciales: Envía usuario y contraseña a LoginServlet mediante POST
  • Manejo de respuesta JSON: Procesa {"success": true/false, "message": "..."}
  • Redirección condicional: Si es exitoso, redirige a jsp/dashboard.jsp
  • Modal de error: Muestra mensaje de error en modal con animación fade-in
  • Cierre de modal: Permite cerrar con botón X o click fuera del modal
  • Reset de formulario: Limpia campos y enfoca usuario después de error
  • Validación de existencia: Verifica que el formulario exista antes de agregar listeners

Servlets Conectados:

Endpoint Método Propósito
LoginServlet POST Autenticar usuario y crear sesión HTTP
Nota: La sesión se valida en el backend con session.getAttribute("usuarioActivo") en páginas protegidas como dashboard.jsp.

Etiquetas: Login Seguridad Sesión AJAX Validación

  • Filtrado con debounce de 300ms
  • Exportación a PDF con jsPDF + AutoTable

  • Uso de Fetch API para peticiones AJAX
