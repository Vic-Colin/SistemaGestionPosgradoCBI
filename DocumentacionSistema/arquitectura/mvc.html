🏗️ Arquitectura MVC

Sistema de Gestión de Posgrado CBI - UAM Azcapotzalco

El sistema sigue el patrón de diseño Modelo-Vista-Controlador (MVC), que separa la lógica de negocio, la interfaz de usuario y el control de flujo de la aplicación.

Diagrama de Flujo MVC

Vista (JSP)
→
Controlador (Servlet)
→
Modelo (DAO/Entity)
→
Base de Datos

📦 Modelo (Model)

Función: Representa la estructura de datos y la lógica de negocio del sistema. Se encarga de gestionar el acceso a la base de datos y las operaciones CRUD (Crear, Leer, Actualizar, Eliminar).

Clases de Entidad (Entities)

Paquete: uam.azc.sistemagestionposgradocbi.modelo

  • Alumno.java - Información académica y administrativa de alumnos
  • Beca.java - Datos de becas CONAHCYT/SECIHTI
  • Profesor.java - Información del núcleo académico
  • Proyecto.java - Proyectos de tesis e investigación
  • Usuario.java - Credenciales y roles de acceso
  • AreaConcentracion.java - Áreas de concentración académica

Características:

  • Atributos privados con getters y setters
  • Implementan Serializable para transporte entre capas
  • Mapeo directo con tablas de la base de datos

Clases DAO (Data Access Object)

Paquete: uam.azc.sistemagestionposgradocbi.DAO

  • AlumnoDAO.java - CRUD de alumnos con JOINs complejos
  • BecaDAO.java - Consultas de becas con filtros dinámicos
  • ProfesorDAO.java - Gestión de profesorado
  • ProyectoDAO.java - Consultas de proyectos de tesis
  • UsuarioDAO.java - Autenticación con SHA-256
  • CrudDAO.java - Interfaz genérica CRUD

Funcionalidades:

  • Conexión a MySQL mediante pool de conexiones (JNDI)
  • PreparedStatements para prevenir SQL Injection
  • Manejo de transacciones (commit/rollback)
  • Consultas con múltiples JOINs
  • Retorno de resultados en formato JSON (Gson)

Utilidades

  • Conexion.java - Obtiene conexiones del pool Tomcat
  • DbCleanupListener.java - Libera recursos JDBC al detener la app

🖼️ Vista (View)

Función: Interfaz gráfica que interactúa con el usuario. Presenta los datos y captura la información del usuario mediante formularios HTML.

Archivos JSP

Ubicación: src/main/webapp/jsp/

  • index.jsp - Página de login con autenticación
  • dashboard.jsp - Menú principal del sistema
  • Alumnos.jsp - Gestión completa de alumnado (CRUD)
  • Becas.jsp - Consulta de becas SECIHTI
  • Profesorado.jsp - Consulta de profesores
  • Proyectos.jsp - Consulta de proyectos de tesis

Tecnologías Frontend

  • HTML5: Estructura semántica de páginas
  • CSS3: Estilos personalizados con diseño responsive
  • JavaScript (ES6): Interactividad y peticiones AJAX
  • JSTL: Etiquetas JSP para iteración y condicionales
  • jsPDF + autoTable: Generación de reportes PDF

Características:

  • Diseño responsive con Bootstrap
  • Tablas dinámicas con filtrado en tiempo real
  • Modales para registro y edición
  • Validación de formularios
  • Exportación de datos a PDF

Archivos JavaScript

Ubicación: src/main/webapp/js/

  • alumnos.js - CRUD completo, filtros, exportación PDF
  • becas.js - Filtrado dinámico de becas
  • profesorado.js - Búsqueda de profesores
  • proyectos.js - Consulta de proyectos
  • index.js - Autenticación y validación de login

🎛️ Controlador (Controller)

Función: Gestiona las solicitudes HTTP del cliente, procesa los datos, invoca el modelo y selecciona la vista apropiada para mostrar resultados.

Servlets

Paquete: uam.azc.sistemagestionposgradocbi.servlets

  • LoginServlet.java - Autenticación de usuarios (POST)
  • LogoutServlet.java - Cierre de sesión
  • AlumnadoServlet.java - CRUD completo de alumnos
  • BecaServlet.java - Consultas de becas con filtros
  • ProfesoradoServlet.java - Búsqueda de profesores
  • ProyectosServlet.java - Consulta de proyectos

Funcionalidades del Controlador

Gestión de Solicitudes:

  • doGet() - Maneja peticiones GET (consultas)
  • doPost() - Maneja peticiones POST (crear/actualizar/eliminar)
  • init() - Inicializa DAO y configuración Gson

Procesamiento de Datos:

  • Recibe parámetros HTTP del frontend
  • Valida y sanitiza datos de entrada
  • Invoca métodos del DAO (modelo)
  • Convierte resultados a JSON usando Gson
  • Establece headers CORS y codificación UTF-8

Seguridad:

  • Validación de sesiones HTTP
  • Hash SHA-256 para contraseñas
  • Protección contra SQL Injection (PreparedStatement)
  • Control de acceso por roles

Flujo de Trabajo Típico

  1. Usuario interactúa con la vista (JSP)
  2. JavaScript envía petición AJAX al servlet
  3. Servlet recibe y valida parámetros
  4. DAO ejecuta consultas SQL en MySQL
  5. Entidades mapean resultados de la BD
  6. Servlet convierte a JSON con Gson
  7. JavaScript recibe JSON y actualiza el DOM
  8. Usuario ve los resultados en la interfaz

🔄 Interacción entre Componentes

Ejemplo: Consulta de Alumnos

  1. Vista: Alumnos.jsp muestra tabla vacía y carga alumnos.js
  2. JavaScript: cargarAlumnos() envía GET a AlumnadoServlet
  3. Servlet: doGet() recibe filtros y llama a AlumnoDAO.buscarPorFiltros()
  4. DAO: Ejecuta SQL con JOINs y retorna List<Alumno>
  5. Servlet: Convierte lista a JSON con Gson.toJson()
  6. JavaScript: Recibe JSON y llama a renderRow() para cada alumno
  7. Vista: Tabla se actualiza dinámicamente sin recargar página
